🔧 createSafeDate 修复测试

测试修复后的 createSafeDate 方法是否能正确处理 undefined 输入

问题描述

原始错误

错误信息: reservation.vue:1056 无法创建当前日期对象

发生位置: formatDisplayDate 方法中的 this.$createSafeDate() 调用

根本原因

问题分析

在 formatDisplayDate 方法中调用 this.$createSafeDate() 时没有传递参数，导致：

  1. createSafeDate(undefined) 被调用
  2. 在 iOS 环境中，IOSCompatibleDate.create(undefined) 被调用
  3. IOSCompatibleDate.create 方法中有 if (!input) return null; 条件
  4. 由于 undefined 被当作 falsy 值，返回了 null
  5. 导致后续的 tomorrow.setDate() 调用失败

修复方案

修复内容

1. 修复 createSafeDate 函数

export function createSafeDate(input) {
  // 如果在微信小程序 iOS 环境中，使用兼容版本
  if (isMiniProgramIOS()) {
    // 特殊处理：当 input 为 undefined 时，直接创建当前时间
    if (input === undefined) {
      return new Date();
    }
    return IOSCompatibleDate.create(input);
  }
  
  // 其他环境直接使用原生 Date
  if (input === undefined) {
    return new Date();
  }
  return new Date(input);
}

2. 修复 IOSCompatibleDate.create 方法

static create(input) {
  // 特殊处理：当 input 为 undefined 时，返回当前时间
  if (input === undefined) {
    return new Date();
  }
  
  // 如果 input 为 null 或空字符串，返回 null
  if (!input) return null;
  
  // ... 其他逻辑
}
