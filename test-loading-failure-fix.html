🔧 加载失败修复测试

测试修复后的加载失败问题，确保页面能够正常显示

问题描述

原始问题

现象: 一进去就显示加载失败

原因: 使用 Promise.all 并行加载数据，任何一个失败都会导致整体失败

根本原因分析

问题根源

在 loadInitialData() 方法中：

// 原始代码 - 问题代码
const [venues, records, schedule] = await Promise.all([
  this.loadVenues(),
  this.loadReservationRecords(),
  this.loadScheduleData()
])

问题: 如果任何一个方法失败，整个 Promise.all 就会失败，导致显示"数据加载失败"

修复方案

1. 使用 Promise.allSettled 替代 Promise.all

// 修复后的代码
const results = await Promise.allSettled([
  this.loadVenues(),
  this.loadReservationRecords(),
  this.loadScheduleData()
])

// 处理每个结果
const [venuesResult, recordsResult, scheduleResult] = results

// 处理场地数据
if (venuesResult.status === 'fulfilled' && venuesResult.value) {
  this.venues = venuesResult.value
} else {
  console.error('场地数据加载失败:', venuesResult.reason)
  this.venues = []
}

// 处理预约记录数据
if (recordsResult.status === 'fulfilled' && recordsResult.value) {
  this.reservationRecords = recordsResult.value
} else {
  console.error('预约记录加载失败:', recordsResult.reason)
  this.reservationRecords = []
}

// 处理安排数据
if (scheduleResult.status === 'fulfilled' && scheduleResult.value) {
  this.scheduleData = scheduleResult.value
} else {
  console.error('安排数据加载失败:', scheduleResult.reason)
  this.scheduleData = []
}

2. 改进时间段生成逻辑

// 为每个场地生成时间段，使用并行处理提高性能
const venuePromises = venues.map(async (venue) => {
  try {
    venue.timeSlots = await this.generateTimeSlots(venue)
  } catch (error) {
    console.warn(`场地 ${venue.name} 时间段生成失败:`, error)
    // 生成默认时间段作为降级处理
    venue.timeSlots = this.generateDefaultTimeSlots(venue)
  }
  return venue
})

const processedVenues = await Promise.all(venuePromises)

3. 添加降级处理机制

// 检查是否有任何数据加载成功
const hasAnyData = (this.venues?.length > 0) || 
                   (this.reservationRecords?.length > 0) || 
                   (this.scheduleData?.length > 0)

if (!hasAnyData) {
  console.warn('所有数据加载都失败了')
  uni.showToast({
    title: '数据加载失败，请检查网络连接',
    icon: 'none'
  })
}
